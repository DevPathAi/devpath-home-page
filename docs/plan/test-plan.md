## Key Interactions to Verify
- 주 CTA "내 실력 진단받기" 클릭 → 앱/대기자 이동 (존재+링크 정확성)
- 리드폼 제출 → 성공/중복/오류 메시지 (Apps Script 모킹)
- 리드폼 프라이버시 가드 → 소스코드/토큰/로그 입력 차단 (기존 form-utils, 회귀 방어)
- traction 티커 → 정상 숫자 / 임계 미만 / fetch 실패 3분기
- LCS 데모 토글 → 맥락 有/無 답변 전환
- 미니진단 3문항 → 샘플 로드맵 스니펫 렌더 (캐닝)
- 스크롤리텔링 → prefers-reduced-motion 시 정적 폴백

## Edge Cases
- **[CRITICAL 회귀]** traction count < 임계값(예 20) 또는 fetch 실패 → "베타 진행 중 — 초대받기" fallback (심사-facing 깨짐 방지)
- JS 비활성 → 본문+CTA+서사 텍스트 유지 (점진적 향상)
- CORS: 라이브 Pages 도메인에서 Apps Script 호출 성공 (로컬만 통과하는 함정)
- 모바일/느린 연결: 위젯 lazy-load, 이미지 CLS 없음
- api.js: timeout, 4xx/5xx 정규화, 빈 응답

## Critical Paths
- 홈 로드(<1s, Lighthouse ≥90) → 주 CTA 클릭 → 이동  [E2E]
- 리드폼 제출 → Sheet 기록 → 성공 UX  [E2E, Apps Script 모킹]
- traction fetch 실패 → fallback 렌더(빈칸/NaN 아님)  [유닛, 최우선]
